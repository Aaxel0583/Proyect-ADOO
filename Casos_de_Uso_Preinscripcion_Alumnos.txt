Casos de Uso: Aplicación de Preinscripción de Alumnos en Línea
Caso de Uso 1: Preinscripción de Alumnos
Descripción: Este caso de uso describe el proceso que un alumno debe seguir para registrarse en la plataforma de preinscripción. El alumno proporciona sus datos personales y adjunta el comprobante de pago para completar el proceso.

Flujo Normal:

Inicio de Sesión: El alumno ingresa a la plataforma de preinscripción utilizando su usuario y contraseña.
Formulario de Preinscripción: Se muestra al alumno un formulario con campos obligatorios para capturar su información personal.
Datos Personales: El formulario solicita al alumno que ingrese su primer apellido, segundo apellido, nombres, sexo, fecha de nacimiento y entidad de nacimiento.
Envío de Formulario: El alumno completa el formulario con sus datos y lo envía a la plataforma.
Generación de CURP: El sistema, con base en la información proporcionada, genera automáticamente la CURP del alumno.
Confirmación de CURP: La plataforma muestra la CURP generada al alumno para su verificación.
Subida de Comprobante de Pago: El alumno adjunta una imagen del comprobante de pago del banco.
Validación de Imagen: El sistema verifica que la imagen del comprobante de pago cumpla con los formatos permitidos (JPG, PNG, PDF).
Finalización de Preinscripción: El alumno completa el proceso de preinscripción.
Registro en Base de Datos: El sistema almacena la información del alumno en la base de datos y muestra un mensaje de éxito confirmando su registro.
Flujos Alternativos:

2.1.1 Datos Incompletos: Si el alumno deja algún campo obligatorio vacío, el sistema muestra un mensaje de error solicitando que complete todos los campos requeridos.
4.1.1 Error en la Generación de CURP: Si el sistema no puede generar la CURP debido a errores en los datos proporcionados, muestra un mensaje indicando que el alumno debe revisar la información ingresada y corregirla.
5.1 Formato Incorrecto de Imagen: Si la imagen del comprobante de pago no cumple con los formatos admitidos (JPG, PNG, PDF), el sistema muestra un mensaje solicitando que se suba una imagen en el formato correcto.
6.1 Problema Técnico en la Subida de Imagen: Si hay un problema técnico durante la subida del comprobante (por ejemplo, falla en la red), el sistema muestra un mensaje de error y solicita que se intente de nuevo más tarde.
8.1 Servidor de Base de Datos No Disponible: Si el servidor de base de datos no está disponible, el sistema muestra un mensaje de error solicitando que se intente de nuevo más tarde.
Caso de Uso 2: Verificación de Pago por Parte del Operador
Descripción: Este caso de uso describe el proceso que un operador debe seguir para verificar la autenticidad de los pagos subidos por los alumnos.

Flujo Normal:

Acceso a la Pantalla de Verificación: El operador ingresa a la pantalla de verificación de pagos.
Búsqueda de Pago: El operador ingresa la CURP del alumno cuyo pago desea verificar.
Visualización de Datos del Alumno: El sistema muestra los datos del alumno, que incluyen su nombre completo, sexo, fecha de nacimiento y entidad de nacimiento.
Visualización del Comprobante de Pago: El sistema muestra la imagen del comprobante de pago subida por el alumno.
Revisión del Comprobante: El operador revisa la imagen del comprobante de pago y verifica su autenticidad.
Confirmación de Pago: El operador marca la casilla de verificación para confirmar que el pago ha sido validado.
Actualización del Estatus de Pago: El sistema actualiza el estatus de pago del alumno a "Pagado".
Flujos Alternativos:

2.1 CURP Inválida o Inexistente: Si la CURP ingresada no es válida o no se encuentra en el sistema, el operador recibe un mensaje de error y se le solicita que verifique la CURP.
4.1 Problema con la Imagen del Comprobante: Si la imagen del comprobante no es visible o es de baja calidad, el operador puede solicitar al alumno que suba un nuevo comprobante de pago.
